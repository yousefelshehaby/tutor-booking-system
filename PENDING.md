# الباقي قبل الإطلاق الفعلي

المشروع شغال محليًا بالكامل (الحجز، الاشتراك الشهري، لوحة الأدمن بكل الأدوار، السوبر أدمن، تصدير Excel). الباقي ثلاث حاجات بس:

## 1. قيم Paymob الناقصة

على جدول `tutors` (المدرّس `يوسف الشحابي` / slug `yousef`) لسه ناقص:

- `paymob_wallet_integration_id` — لازم تعمل تكامل "Mobile Wallet" في لوحة Paymob وتاخد الـ Integration ID
- `paymob_fawry_integration_id` — عندك تكامل CASH اتعمل بالفعل في لوحة Paymob، محتاج بس تاخد الـ Integration ID بتاعه وتديهولي
- `paymob_iframe_id` — من Paymob → Settings → Payment Integrations → Iframes، لازم تعمل iframe مرتبط بتكامل البطاقة (Card) وتاخد الـ ID بتاعه

من غير `paymob_iframe_id` تحديدًا، حتى الدفع بالبطاقة مش هيشتغل (مش بس المحفظة وفوري).

## 2. النشر على Vercel + إعدادات Paymob

- المشروع مرفوع على GitHub بس لسه مانزلش على Vercel فعليًا (وصلنا لشاشة الإعداد بس مفيش تأكيد إن "Deploy" اتضغط)
- بعد النشر: لازم تحدّث في لوحة Paymob:
  - **Transaction Processed Callback** (الـ webhook) → `https://<الدومين بتاعك>/api/webhooks/paymob`
  - **Transaction redirection URL** (لكل تكامل: بطاقة/محفظة) → `https://<الدومين بتاعك>/yousef/payment/result`
- تفاصيل كل متغيرات البيئة المطلوبة في Vercel موجودة في [README.md](README.md)

## 3. تشيكليست اختبار الموقع الحي (بعد النشر)

- [ ] فتح الموقع من الدومين الحقيقي (مش localhost) وعمل حجز كامل
- [ ] دفع تجريبي ببطاقة Paymob التجريبية والتأكد إن الحجز اتحول لـ "تم الدفع" فعليًا من قاعدة البيانات (مش من صفحة الـ redirect بس)
- [ ] تجربة الدفع بمحفظة إلكترونية وفوري
- [ ] تجربة "حجز بدون دفع الآن" والتأكد من تحذير الـ 48 ساعة
- [ ] دخول admin1123 (سوبر أدمن) والتأكد إن كل الصلاحيات شغالة زي ما هي محليًا
- [ ] دخول حساب مدرّس عادي والتأكد إنه مايشوفش غير بياناته
- [ ] تجربة تصدير Excel (الحجوزات + المدفوعات الشهرية)
